Sexual Integrity Curriculum

16-Week Program Application - Technical Documentation

Last Updated: January 28, 2026

1. Project Overview

A comprehensive 16-week Sexual Integrity curriculum program web application designed for therapists and clients. The application features role-based access (Admin, Therapist, Client), weekly lessons combining CBT (weeks 1-8) and ACT (weeks 9-16) content, time-based week unlocking, daily self-monitoring tools, and integrated Stripe payment processing.

Key Features

  • Role-based access control (Admin, Therapist, Client)
  • 16-week curriculum with CBT and ACT therapeutic content
  • Time-based week unlocking (7-day intervals)
  • Daily self-monitoring and check-in tools
  • Therapist-client assignment and progress monitoring
  • Stripe payment integration for subscriptions and per-week payments
  • AI-powered encouragement and technique reminders
  • Password reset via email
  • Crisis resources component

3. Technology Stack

Frontend

  • Framework: React 18 with Vite
  • Routing: Wouter (lightweight client-side routing)
  • State Management: TanStack React Query for server state, React Context for auth
  • Styling: Tailwind CSS with shadcn/ui component library
  • Form Handling: React Hook Form with Zod validation

Backend

  • Framework: Express.js with TypeScript
  • Authentication: Passport.js with Local Strategy
  • Session Storage: PostgreSQL-backed via connect-pg-simple
  • Payments: Stripe integration
  • Email: Gmail integration for password resets
  • AI: OpenAI-compatible API for encouragement features

Database

  • Database: PostgreSQL
  • ORM: Drizzle ORM
  • Schema Validation: drizzle-zod

5. API Endpoints

Authentication

  • POST /api/auth/login - User login
  • POST /api/auth/logout - User logout
  • GET /api/auth/me - Get current user
  • POST /api/auth/register/therapist - Therapist registration
  • POST /api/auth/register/client - Client registration
  • POST /api/auth/forgot-password - Request password reset
  • POST /api/auth/reset-password - Reset password with token
  • POST /api/auth/change-password - Change password (logged in)

Client Progress

  • GET /api/progress/weeks - Get week completion status
  • POST /api/progress/weeks/:weekNumber/complete - Mark week complete
  • GET /api/progress/checkin/:dateKey - Get daily check-in
  • PUT /api/progress/checkin/:dateKey - Save daily check-in
  • GET /api/progress/reflection/:weekNumber - Get week reflection
  • PUT /api/progress/reflection/:weekNumber - Save week reflection
  • GET /api/progress/homework/:week - Get homework completion
  • PUT /api/progress/homework/:week - Save homework completion

Therapist

  • GET /api/therapist/clients - Get assigned clients
  • GET /api/therapist/clients/:clientId/progress - Get client progress
  • GET /api/therapist/clients/:clientId/checkins - Get client check-ins
  • GET /api/therapist/clients/:clientId/reflections - Get client reflections
  • POST /api/therapist/feedback - Submit feedback to client
  • GET /api/therapist/feedback/:clientId - Get feedback for client

Admin

  • GET /api/admin/therapists - List all therapists
  • GET /api/admin/clients - List all clients
  • GET /api/admin/unassigned-clients - List unassigned clients
  • POST /api/admin/assign-client - Assign client to therapist
  • PATCH /api/admin/clients/:clientId - Update client
  • DELETE /api/admin/clients/:clientId - Delete client
  • PATCH /api/admin/therapists/:therapistId - Update therapist (fee waiver)
  • POST /api/admin/therapist-departure/:therapistId - Handle therapist departure
  • GET /api/admin/revenue-by-therapist - Revenue tracking

Payments (Stripe)

  • GET /api/stripe/config - Get Stripe price IDs
  • POST /api/payments/checkout/subscription - Create therapist subscription
  • POST /api/payments/checkout/week - Create week payment checkout
  • GET /api/payments/subscription - Get subscription status
  • GET /api/payments/week/:weekNumber/status - Check week payment status
  • POST /api/stripe/webhook - Stripe webhook handler

AI Features

  • GET /api/ai/encouragement?week=N - Get AI encouragement
  • GET /api/ai/technique?week=N&technique=name - Get technique reminder

8. Setup Instructions

1. Install Dependencies

npm install

2. Set Up Database

# Create PostgreSQL database and set DATABASE_URL
npm run db:push

3. Configure Environment Variables

Create a .env file with all required variables listed in section 6.

4. Seed Stripe Products

npx tsx scripts/seed-stripe-products.ts

5. Run Development Server

npm run dev

Server runs on port 5000 (Express + Vite)

6. Production Build

npm run build
npm start

9. Key Technical Notes

  1. Stripe Webhook: Must be registered BEFORE express.json() middleware
  2. Week Unlocking: Week 1 available immediately; subsequent weeks unlock 7 days after previous
  3. Payment Verification: Checkout sessions verified with Stripe API before recording payment
  4. Fee Waivers: Admin can waive all fees (allFeesWaived flag) or per-week waivers
  5. Session Storage: PostgreSQL-backed sessions via connect-pg-simple
  6. AI Boundaries: AI limited to encouragement only - no crisis intervention or medical advice
